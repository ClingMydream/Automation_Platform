## 2026-07-06 补充：问题定位层完成

本次补齐平台主链路中的“问题定位”环节：

```text
测试对象 -> 测试能力 -> 执行调度 -> 结果采集 -> 问题定位 -> 质量分析 -> 报告输出 -> 集成开放
```

新增目录：

```text
backend/app/modules/problem_diagnosis/
frontend/src/modules/04-problem-diagnosis/
```

新增接口：

```text
GET    /api/v1/problem-findings
POST   /api/v1/problem-findings
PUT    /api/v1/problem-findings/{finding_id}
DELETE /api/v1/problem-findings/{finding_id}
POST   /api/v1/problem-findings/from-result/{result_id}
```

使用方式：

```text
1. 先在“结果中心”或“执行记录”产生失败结果。
2. 打开左侧菜单“问题定位”。
3. 在左侧失败结果列表点击“生成定位”。
4. 在右侧问题定位记录中补充根因、复现步骤、责任人和处理建议。
5. 修复完成后把状态改为“已修复”。
```
