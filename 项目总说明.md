## 2026-07-06 补充：测试环境管理与健康检查

本次补齐“测试环境管理”可操作入口，让平台不只保存项目，还能维护环境并检查环境是否可访问。

新增能力：

```text
项目页新增环境配置表单
项目页新增环境列表
环境支持新增、修改、删除
环境支持一键健康检查

```

涉及接口：

```text
GET    /api/environments
POST   /api/environments
PUT    /api/environments/{environment_id}
DELETE /api/environments/{environment_id}
POST   /api/v1/environments/{environment_id}/health-check
```

使用方式：

```text
1. 打开“项目”菜单。
2. 在“新建环境”中选择所属项目。

5. 保存后在“环境列表”点击“检查”，确认该环境可访问。
```

## 2026-07-07 补充：接口批量任务执行与结果沉淀

本次补齐 MVP 中的“接口批量执行任务”闭环，让测试任务不再只是保存配置，而是可以真正调度一组接口用例。

新增能力：

```text
测试任务页新增执行环境选择
测试任务页新增接口用例集合多选
API 类型任务保存时自动写入 config.api_case_ids
点击任务执行后创建一个 execution batch
平台为批次里的每个接口用例创建一条 TestRun 并推入 Redis 队列
worker 执行完成后把单条结果写入结果中心 test_results
worker 自动刷新批次总数、通过数、失败数、跳过数、耗时和最终状态
```

兼容策略：

```text
旧任务仍可使用 config.case_ids
旧执行记录不受影响
test_runs 新增 batch_id、task_id 两个可空字段
只放宽环境列表和接口用例列表的读取权限给测试任务页，不放宽新增/修改/删除权限
```

操作方式：

```text
1. 打开“接口测试”，先准备接口用例。
2. 打开“测试任务”，任务类型选择“接口”。
3. 选择执行环境和接口用例集合。
4. 保存任务。
5. 在任务列表点击“执行”。
6. 打开“结果中心”，查看批次和每条接口执行结果。
```

涉及文件：

```text
backend/app/modules/test_tasks/
backend/app/modules/result_center/

backend/app/models/entities.py
worker/app/infrastructure/db.py
```


## 10.11 2026-07-07 执行调度层增强：定时任务

改动内容：

```text
测试任务页新增定时 cron 输入框和列表展示。
worker 新增轻量定时扫描器，默认每 30 秒扫描一次。
命中 cron 的 API + platform 任务会自动创建 execution batch。
定时批次会复用现有 API 批量执行链路，自动生成 TestRun 并推入 Redis 队列。
同一分钟内不会重复触发同一个任务。
```

支持范围：

```text
第一版自动调度仅支持 task_type=api 且 runner_type=platform 的任务。
UI、性能、脚本、兼容性任务仍建议由外部 runner 或 CI 触发后回传结果。
cron 表达式使用 5 段格式：分钟 小时 日期 月份 星期。
支持 *、*/N、单值、逗号列表和区间，例如 */10 * * * *。
```

配置项：

```text
SCHEDULER_ENABLED=true
SCHEDULER_POLL_SECONDS=30
```

操作方式：

```text
1. 打开“测试任务”。
2. 创建 API 类型任务，执行来源保持 platform。
3. 选择接口用例集合。
4. 在“定时 cron”填写 */10 * * * *。
5. 保存并保持任务启用。
6. worker 到点后会自动创建定时批次，可在“结果中心”和“测试报告”查看。
```
