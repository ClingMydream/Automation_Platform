## 2. 登录信息

平台管理员账号：

```text
admin
```

平台管理员密码不写入本文档，也不要提交到 GitHub。当前部署环境的管理员密码配置在服务器：

```text
/opt/automation-platform/.env
```

对应字段：

```text
ADMIN_USERNAME=admin
ADMIN_PASSWORD=请在服务器 .env 中查看或重新设置
```

查看方式：

```bash
cd /opt/automation-platform
grep '^ADMIN_' .env
```

修改管理员密码后，需要重启后端服务：

```bash
cd /opt/automation-platform
docker compose up -d backend
```

重要安全说明：

- 不要把服务器密码、GitHub 密码、平台管理员密码写进 GitHub。
- 已经在聊天、文档或截图中出现过的密码，都应该视为已经泄露，建议尽快更换。
- GitHub 推送建议使用 SSH key，不使用账号密码。
- 服务器建议开启 SSH key 登录后关闭 root 密码登录。

## 4. 项目技术栈

前端：

```text
React + Vite + Ant Design
```

后端：

```text
FastAPI + SQLAlchemy + MySQL + Redis
```

执行器：

```text
接口自动化：Python 3 + pytest + requests + allure
UI 自动化：Python 3 + pytest + Playwright + allure
```

部署：

```text
Docker Compose + Nginx
```

## 5. 代码分层总览

```text
frontend/   前端页面和浏览器端工具
backend/    后端接口、认证、权限、数据库模型、业务服务
worker/     自动化测试执行器
deploy/     部署脚本和 Nginx 配置
docs/       小白部署、使用、改造文档
```

```text

backend/app/modules/<模块名>/schemas.py   模型层：只属于该模块的请求/响应模型
```
