## 跨电脑继续开发的方式

### 1. 拉代码

```bash
git clone git@github.com:ClingMydream/Automation_Platform.git
cd Automation_Platform
git checkout main
git pull origin main
```

### 2. 优先阅读这些文档

- `docs/改造进度同步记录.md`
- `项目总说明.md`
- `docs/代码改造指南.md`
- `docs/模块化架构说明.md`
- `docs/接口文档与JMeter压测说明.md`
- `docs/小白部署与使用手册.md`

### 3. 本地启动

```bash
cp .env.example .env
docker compose up -d --build
```

如果只是前端开发或后端开发，也可以分别进入对应目录单独启动。

## 同步原则

- 代码和文档一起提交，避免“代码变了，说明没跟上”。
- 不把 `.env`、私钥、数据库数据、附件产物提交到 Git。
- 每次做完一个阶段性能力，都在本文件追加一次进度说明。

## 下一次提交建议

后续每次继续改造时，建议提交信息直接写清楚能力名称，例如：

- `Add live UI execution viewer`
