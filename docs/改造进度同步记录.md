## 2026-07-09 补充：Mock 服务公开响应出口

本次完成：

- 新增公开 Mock 响应服务模块 `backend/app/modules/mock_service/`。
- 已配置的 Mock 规则现在可以通过 `/api/mock/{path}` 返回真实 HTTP 响应。
- Mock 响应支持按请求方法和路径匹配。
- Mock 响应支持配置状态码、响应头、响应体和延迟。
- Mock 出口不需要登录，方便前端页面、接口调试或自动化用例直接调用。
- 响应头会过滤 `Content-Length`、`Transfer-Encoding`、`Connection` 等不安全传输头。

使用方式：

```text
1. 在“测试能力 / Mock 规则”中新增规则：
   method = GET
   path = /api/demo
   status_code = 200
   response_body = {"message":"mock ok"}

2. 访问：
   GET /api/mock/api/demo
```

影响范围：

- backend: `backend/app/modules/mock_service/`、`backend/app/api/routes.py`、`backend/app/main.py`
- frontend: 无页面结构变更
