# 自动化测试平台代码改造指南

这份文档给想参与代码编写的人看。目标不是一次记住所有代码，而是知道“要改某个功能时，先打开哪个文件”。

## 1. 项目代码在哪里

本机目录：

```text
D:\Automation_Platform
```

常用目录：

```text
frontend/                 前端页面，React + Vite
frontend/src/main.jsx     前端应用总入口，只负责登录态、菜单和模块切换
frontend/src/modules/     前端业务模块，每个功能一个文件夹
frontend/src/shared/      前端通用工具函数，适合单独改造
frontend/src/styles/      前端样式

backend/                  后端接口，FastAPI
backend/app/api/routes.py 后端接口总入口
backend/app/core/         后端通用能力：认证、配置、权限菜单、目标地址安全校验
backend/app/models/       数据库表结构
backend/app/schemas/      接口请求和响应字段定义

worker/                   执行接口测试和 UI 自动化
worker/app/modules/       自动化执行模块，接口/UI 自动化代码侧实现
deploy/                   部署脚本、Nginx 配置
docs/                     文档
```

## 3. 前端通用工具怎么改

为了让模块更独立，已经把通用逻辑拆到 `frontend/src/shared/`。

| 文件 | 负责内容 | 适合怎么改 |
| --- | --- | --- |
| `codec.js` | URL、Base64、Unicode、HTML、Hex、JSON 字符串转码 | 新增转码类型时改这里，再到 `CodecPanel` 加下拉选项 |
| `jsonTools.js` | JSON 格式化、压缩、对比算法 | 新增“忽略字段”“数组按字段匹配”等能力 |
| `fileTransfer.jsx` | 文件类型识别、图片/视频预览、浏览器下载 | 增加更多文件预览方式 |
| `formatters.js` | 时间、耗时、文件大小、状态颜色 | 调整显示格式 |
| `html.js` | HTML 转义 | 一般不用改 |
| `reportExport.js` | 导出 HTML 测试报告 | 调整报告模板、报告样式 |

## 5. 权限菜单在哪里改

菜单权限统一在：

```text
backend/app/core/menu.py
```

新增一个普通模块时，通常要改 3 个地方：

1. 在 `backend/app/core/menu.py` 的 `MENU_OPTIONS` 增加一项。
2. 在 `frontend/src/modules/` 新增一个模块文件夹和页面组件。
3. 在 `frontend/src/main.jsx` 的 `allMenuItems` 增加左侧菜单。
4. 在 `frontend/src/main.jsx` 的内容区域增加 `{tab === 'xxx' && <你的组件 />}`。

如果这个模块需要后端接口，还要在 `backend/app/api/routes.py` 增加接口，并用 `require_menu("你的菜单key")` 做权限校验。

## 6. 新增一个页面模块的推荐步骤

假设要新增“通知中心”：

1. 后端菜单加权限：

```python
{"key": "notifications", "label": "通知中心"}
```

2. 前端新增组件：

```jsx
function NotificationsPanel() {
  return <Card title="通知中心">这里写页面内容</Card>;
}
```

3. 前端左侧菜单加一项：

```jsx
{ key: 'notifications', icon: <BellOutlined />, label: '通知中心' }
```

4. 前端内容区域加映射：

```jsx
{tab === 'notifications' && <NotificationsPanel />}
```

5. 如果需要接口，在 `routes.py` 增加：

```python
@router.get("/notifications")
def list_notifications(_: AuthContext = Depends(require_menu("notifications"))):
    return []
```

## 8. 写代码时的简单规则

- 页面组件只管页面展示和按钮交互。
- 通用算法放到 `frontend/src/shared/`。
- 后端权限菜单只改 `backend/app/core/menu.py`。
- 后端接口要加权限校验，不要裸奔。
- 不要把密码、token、服务器密钥写进代码。
- 改一个模块时，尽量只动这个模块相关文件。
- 每次改完先构建，再提交。

## 9. 当前还可以继续优化的方向

现在已经做了模块化改造：前端页面按模块文件夹拆分，worker 自动化执行按接口/UI 自动化模块拆分，菜单权限独立。

后续如果要进一步工程化，可以继续做：

- 把 `routes.py` 拆成多个后端 router 文件。
- 给每个模块加单元测试。
- 把接口请求封装成前端 `services/`。
- 增加 ESLint 和 Prettier，统一代码格式。
