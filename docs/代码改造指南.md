# 自动化测试平台代码改造指南

这份文档给想参与代码编写的人看。目标不是一次记住所有代码，而是知道“要改某个功能时，先打开哪个文件”。

## 1. 项目代码在哪里

本机目录：

```text
D:\Automation_Platform
```

常用目录：

```text
frontend/                 前端页面，React + Vite
frontend/src/main.jsx     前端应用总入口，只负责登录态、菜单和模块切换
frontend/src/modules/     前端业务模块，按菜单名分文件夹
frontend/src/shared/      前端通用工具函数，适合多个页面复用
frontend/src/styles/      前端样式

backend/                  后端接口，FastAPI
backend/app/api/routes.py 后端接口聚合入口
backend/app/modules/      后端业务模块，每个模块有自己的 router.py
backend/app/core/         后端通用能力：认证、配置、权限菜单、目标地址安全校验
backend/app/models/       数据库表结构
backend/app/schemas/      接口请求和响应字段定义

worker/                   执行接口测试和 UI 自动化
worker/app/modules/       自动化执行模块，接口/UI 自动化代码侧实现
deploy/                   部署脚本、Nginx 配置
docs/                     文档
```

## 2. 前端模块怎么找

前端业务页面已经按菜单拆到 `frontend/src/modules/`。每个菜单一个文件夹，每个文件夹里再分“功能”和“UI”。

| 页面 | UI 目录 | 功能说明目录 | 主要负责 |
| --- | --- | --- | --- |
| 登录 | `frontend/src/modules/登录/登录UI/` | `frontend/src/modules/登录/登录功能/` | 登录表单、token 保存、登录过期跳转 |
| 项目 | `frontend/src/modules/项目/项目UI/` | `frontend/src/modules/项目/项目功能/` | 项目和环境管理 |
| 接口测试 | `frontend/src/modules/接口测试/接口测试UI/` | `frontend/src/modules/接口测试/接口测试功能/` | 接口用例维护、JSON 请求和断言说明、执行入口 |
| UI 测试 | `frontend/src/modules/UI测试/UI测试UI/` | `frontend/src/modules/UI测试/UI测试功能/` | UI 步骤维护、实时执行窗口入口 |
| 文件快传 | `frontend/src/modules/文件快传/文件快传UI/` | `frontend/src/modules/文件快传/文件快传功能/` | 文件上传、二维码、临时下载 |
| 图片工具 | `frontend/src/modules/图片工具/图片工具UI/` | `frontend/src/modules/图片工具/图片工具功能/` | 图片生成、裁剪、格式转换 |
| JSON 工具 | `frontend/src/modules/JSON工具/JSON工具UI/` | `frontend/src/modules/JSON工具/JSON工具功能/` | JSON 格式化、压缩、对比 |
| 转码工具 | `frontend/src/modules/转码工具/转码工具UI/` | `frontend/src/modules/转码工具/转码工具功能/` | URL、Base64、Unicode 等转码 |
| 执行记录 | `frontend/src/modules/执行记录/执行记录UI/` | `frontend/src/modules/执行记录/执行记录功能/` | 任务列表、执行详情、日志、截图 |
| 测试报告 | `frontend/src/modules/测试报告/测试报告UI/` | `frontend/src/modules/测试报告/测试报告功能/` | 报告列表、报告详情、HTML 导出 |
| 用户管理 | `frontend/src/modules/用户管理/用户管理UI/` | `frontend/src/modules/用户管理/用户管理功能/` | 用户创建、菜单权限配置 |
| 实时执行窗口 | `frontend/src/modules/实时执行窗口/实时执行窗口UI/` | `frontend/src/modules/实时执行窗口/实时执行功能/` | UI 自动化运行过程展示 |

## 3. 为什么还会有 shared 目录

`shared` 是“多个模块都会用到”的公共工具区。比如每个页面都要请求后端，所以请求封装在 `frontend/src/shared/apiClient.js`，而不是每个菜单目录复制一份。

常见文件：

| 文件 | 负责内容 |
| --- | --- |
| `apiClient.js` | 所有前端请求后端的统一入口 |
| `codec.js` | URL、Base64、Unicode、HTML、Hex、JSON 字符串转码 |
| `jsonTools.js` | JSON 格式化、压缩、对比算法 |
| `fileTransfer.jsx` | 文件类型识别、图片/视频预览、浏览器下载 |
| `formatters.js` | 时间、耗时、文件大小、状态颜色 |
| `reportExport.js` | 导出 HTML 测试报告 |

## 4. 后端模块怎么找

后端入口是 `backend/app/api/routes.py`。真正的业务接口放在 `backend/app/modules/`。

| 后端模块文件 | URL 前缀 | 主要负责 |
| --- | --- | --- |
| `backend/app/modules/auth/router.py` | `/auth` | 登录、退出、当前用户 |
| `backend/app/modules/users/router.py` | `/users`、`/menu-options` | 用户增删改查、菜单权限 |
| `backend/app/modules/projects/router.py` | `/projects`、`/environments` | 项目和环境 |
| `backend/app/modules/api_testing/router.py` | `/api-cases` | 接口用例保存、修改、删除 |
| `backend/app/modules/ui_testing/router.py` | `/ui-cases` | UI 步骤保存、修改、删除 |
| `backend/app/modules/file_transfer/router.py` | `/file-transfers` | 临时文件上传、下载、预览 |
| `backend/app/modules/image_tools/router.py` | `/image-tools` | 图片生成、裁剪、格式转换 |
| `backend/app/modules/runs/router.py` | `/runs` | 创建执行任务、查询执行记录 |
| `backend/app/modules/reports/router.py` | `/reports` | 查询报告列表和详情 |

## 5. 自动化代码侧怎么找

接口自动化执行代码：

```text
worker/app/modules/api_automation/
```

这里使用：

```text
Python 3 + pytest + requests + allure
```

UI 自动化执行代码：

```text
worker/app/modules/ui_automation/
```

这里使用：

```text
Python 3 + pytest + Playwright + allure
```

前端页面只负责“让用户填写和点击”，真正执行自动化的代码放在 worker。

## 6. 新增一个页面模块的推荐步骤

假设要新增“通知中心”：

1. 在 `backend/app/core/menu.py` 增加菜单权限：

```python
{"key": "notifications", "label": "通知中心"}
```

2. 在前端新增目录：

```text
frontend/src/modules/通知中心/
  通知中心功能/
    README.md
  通知中心UI/
    NotificationsPanel.jsx
    README.md
```

3. 在 `frontend/src/main.jsx` 引入组件，并把它加入菜单切换。

4. 如果需要后端接口，在 `backend/app/modules/notifications/router.py` 新增接口，再到 `backend/app/api/routes.py` include 这个 router。

## 7. 修改代码前的小建议

- 先改一个小点，再运行构建或重启服务验证。
- 改 UI 先看 `某某UI` 目录。
- 改业务逻辑先看 `某某功能/README.md`，再去后端或 worker。
- 不要把密码、`.env`、数据库文件、截图产物提交到 GitHub。
