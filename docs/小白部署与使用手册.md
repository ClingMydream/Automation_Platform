# 自动化测试平台小白部署与使用手册

本文档面向 0 代码基础用户，按步骤操作即可完成部署、使用和后续升级。

## 1. 这个平台能做什么

自动化测试平台用于把重复测试流程放到网页里管理。

- 接口测试：填写接口地址、请求方法、请求头、请求体和断言，点击执行后自动检查结果。
- UI 测试：用低代码步骤描述网页操作，例如打开网页、点击按钮、输入文字、检查页面文字。
- 执行记录：每次执行都会保存状态、耗时、错误信息和报告。
- 安全限制：默认只允许测试公网 HTTP/HTTPS 地址，禁止访问服务器本机、内网和云元数据地址。

## 2. 项目组成

- `frontend/`：网页控制台。
- `backend/`：后端接口服务，负责登录、保存用例、下发执行任务。
- `worker/`：测试执行器，负责真正访问接口或浏览器页面。
- `docker-compose.yml`：一键启动 MySQL、Redis、后端、Worker 和前端。
- `.env.example`：配置模板。真实密码要写到 `.env`，不要提交到 GitHub。

## 3. 第一次部署到云服务器

### 3.1 登录服务器

在 Windows 终端中执行：

```powershell
ssh root@111.229.178.141
```

如果提示输入密码，就输入服务器密码。输入密码时屏幕不显示字符，这是正常现象。

### 3.2 安装基础软件和安全工具

进入项目目录后执行：

```bash
bash deploy/scripts/init_server.sh
```

这个脚本会安装：

- Git：用于拉取 GitHub 代码。
- Docker 和 Docker Compose：用于运行平台。
- firewalld：服务器防火墙。
- fail2ban：防止别人不断猜 SSH 密码。如果系统仓库没有这个包，脚本会继续部署，并保留防火墙保护。

脚本会开放 22 和 80 端口。22 用于 SSH 登录，80 用于浏览器访问平台。

### 3.3 配置腾讯云安全组

如果浏览器打不开平台，需要在腾讯云控制台检查安全组：

1. 打开云服务器实例页面。
2. 找到安全组或防火墙规则。
3. 添加入站规则：TCP，端口 `80`，来源 `0.0.0.0/0`。
4. SSH 的 `22` 端口也要保留。

### 3.4 创建配置文件

在服务器项目目录 `/opt/automation-platform` 中执行：

```bash
cp .env.example .env
```

然后编辑 `.env`：

```bash
vi .env
```

必须修改这些值：

- `APP_SECRET_KEY`：改成一串长随机字符。
- `ADMIN_PASSWORD`：改成管理员登录密码。
- `MYSQL_PASSWORD`：改成数据库普通用户密码。
- `MYSQL_ROOT_PASSWORD`：改成数据库 root 密码。

注意：`.env` 里面都是秘密，不要上传到 GitHub，也不要截图发给别人。

### 3.5 启动平台

在 `/opt/automation-platform` 目录执行：

```bash
bash deploy/scripts/deploy.sh
```

查看服务状态：

```bash
docker compose ps
```

如果看到大部分服务是 `running` 或 `healthy`，说明启动成功。

### 3.6 浏览器访问

打开：

```text
http://111.229.178.141
```

默认账号是：

```text
admin
```

密码是 `.env` 里设置的 `ADMIN_PASSWORD`。

## 4. 怎么使用平台

### 4.1 创建项目

1. 登录平台。
2. 进入“项目”。
3. 输入项目名称，例如“官网接口测试”。
4. 点击保存。

项目用于把不同系统的测试用例分组。

### 4.2 创建接口测试用例

1. 进入“接口测试”。
2. 选择项目。
3. 填写用例名称。
4. 选择请求方法，例如 `GET` 或 `POST`。
5. 填写完整 URL，例如 `https://example.com`。
6. 请求头 JSON 不需要时填写 `{}`。
7. 按需要填写请求体。
8. 填写断言，例如状态码 `200`，或响应中必须包含某段文字。
9. 点击保存。
10. 在用例列表中点击“执行”。

### 4.3 创建 UI 测试用例

UI 用例用 JSON 写步骤。示例：

```json
[
  {"action": "goto", "value": "https://example.com"},
  {"action": "assert_text", "value": "Example Domain"},
  {"action": "screenshot"}
]
```

支持的动作：

- `goto`：打开网页，`value` 填完整网址。
- `click`：点击元素，`target` 填 CSS 选择器。
- `fill`：输入文字，`target` 填输入框选择器，`value` 填要输入的文字。
- `wait`：等待，`value` 填毫秒数。
- `assert_text`：检查页面中是否出现文字，`value` 填文字。
- `screenshot`：截图。

第一版不允许上传脚本，这是为了避免平台被人当成远程执行工具。

### 4.4 查看执行结果

进入“执行记录”，可以看到：

- `queued`：排队中。
- `running`：执行中。
- `passed`：通过。
- `failed`：失败。

页面下方会显示最新一次执行的 JSON 报告。

## 5. 日常维护

### 5.1 查看服务状态

```bash
cd /opt/automation-platform
docker compose ps
```

### 5.2 查看日志

查看后端日志：

```bash
docker compose logs -f backend
```

查看 Worker 日志：

```bash
docker compose logs -f worker
```

### 5.3 重启平台

```bash
docker compose restart
```

### 5.4 停止平台

```bash
docker compose down
```

### 5.5 备份数据库

```bash
docker compose exec mysql mysqldump -u root -p automation_platform > backup.sql
```

执行后会要求输入 `.env` 里的 `MYSQL_ROOT_PASSWORD`。

## 6. 后续升级改造

### 6.1 修改网页

网页代码在 `frontend/`。

常见修改：

- 改页面文字：看 `frontend/src/main.jsx`。
- 改颜色和间距：看 `frontend/src/styles/app.css`。
- 增加页面：先在 `main.jsx` 增加标签页，再增加对应组件。

修改后重新部署：

```bash
docker compose up -d --build frontend
```

### 6.2 修改后端接口

后端代码在 `backend/app/`。

常见文件：

- `api/routes.py`：接口地址和处理逻辑。
- `models/entities.py`：数据库表结构。
- `schemas/entities.py`：接口输入输出格式。
- `core/target_guard.py`：测试目标安全限制。

修改后重新部署：

```bash
docker compose up -d --build backend worker
```

### 6.3 修改 UI 测试能力

Worker 代码在 `worker/app/worker.py`。

如果想支持新的 UI 动作，例如选择下拉框，可以在 `run_ui_case` 里增加一个 `elif action == "select"` 分支。

修改后重新部署：

```bash
docker compose up -d --build worker
```

## 7. 安全注意事项

### 7.1 立刻更换已经泄露过的密码

如果密码、Token、私钥出现在聊天、截图或文档中，都应该立刻更换。

### 7.2 不要提交秘密文件

不要把这些文件上传到 GitHub：

- `.env`
- 私钥文件，例如 `id_rsa`
- 数据库备份
- 测试截图中包含敏感信息的文件

项目已经用 `.gitignore` 默认排除了这些文件。

### 7.3 关闭 SSH 密码登录

确认 SSH key 登录成功后，在服务器执行：

```bash
bash deploy/scripts/harden_ssh_after_key_login.sh
```

这样别人就不能通过猜密码登录服务器。

### 7.4 不要开放数据库端口

MySQL 和 Redis 只在 Docker 内网访问，不应该开放到公网。

公网只需要开放：

- `22`：SSH
- `80`：网页访问

### 7.5 如何降低木马挖矿风险

- 使用强密码和 SSH key。
- 开启 firewalld，只开放必要端口。
- 开启 fail2ban。
- 不上传和运行陌生脚本。
- 平台不支持任意脚本上传。
- Worker 容器不挂载 Docker socket，不能控制宿主机 Docker。
- 定期执行系统更新：

```bash
dnf update -y
```

## 8. 常见问题

### 浏览器打不开平台

检查三件事：

1. 腾讯云安全组是否开放 TCP 80。
2. 服务器防火墙是否开放 HTTP：

```bash
firewall-cmd --list-all
```

3. Docker 服务是否启动：

```bash
docker compose ps
```

### 登录失败

检查 `.env` 中的：

```text
ADMIN_USERNAME
ADMIN_PASSWORD
```

修改后重启后端：

```bash
docker compose restart backend
```

### 用例执行一直 queued

说明 Worker 没有正常消费任务，查看日志：

```bash
docker compose logs -f worker
```

### 测试内网地址被拒绝

这是安全策略。公网部署默认只允许测试公网 HTTP/HTTPS 地址，防止被滥用扫描内网。

如果未来确实需要测试内网地址，应改成白名单机制，而不是直接放开所有内网访问。
