# 模块化架构说明

本项目按“菜单模块独立、功能实现和界面展示分离”的方式组织。菜单在页面上仍显示中文，代码目录使用英文，减少 Windows、Git、Docker 和服务器脚本里的编码问题。

## 前端目录

前端业务模块目录：

```text
frontend/src/modules/
```

当前结构示例：

```text
frontend/src/modules/
  api-testing/
    feature/
      apiCaseFeature.js
      README.md
    ui/
      ApiCasePanel.jsx
      README.md
  ui-testing/
    feature/
      uiCaseFeature.js
      README.md
    ui/
      UiCasePanel.jsx
      README.md
  run-history/
    feature/
      runHistoryFeature.js
      README.md
    ui/
      RunsPanel.jsx
      RunDetail.jsx
      README.md
```

每个菜单模块都有两个子目录：

| 子目录 | 作用 |
| --- | --- |
| `feature/` | 放功能实现、数据转换、接口调用、统计、筛选、校验逻辑。 |
| `ui/` | 放页面组件，主要负责按钮、表格、表单、弹窗、提示文案和页面样式。 |

前端通用能力统一放在：

```text
frontend/src/shared/
```

例如请求封装、JSON 工具、转码工具、报告导出、文件预览等。多个页面都会用到的能力不要复制到每个模块里，优先放到 `shared`。

## 模块对照

| 页面菜单 | 前端模块 |
| --- | --- |
| 登录 | `frontend/src/modules/auth/` |
| 项目 | `frontend/src/modules/projects/` |
| 接口测试 | `frontend/src/modules/api-testing/` |
| UI 测试 | `frontend/src/modules/ui-testing/` |
| 文件快传 | `frontend/src/modules/file-transfer/` |
| 图片工具 | `frontend/src/modules/image-tools/` |
| JSON 工具 | `frontend/src/modules/json-tools/` |
| 转码工具 | `frontend/src/modules/codec-tools/` |
| 执行记录 | `frontend/src/modules/run-history/` |
| 测试报告 | `frontend/src/modules/test-reports/` |
| 用户管理 | `frontend/src/modules/user-management/` |
| 实时执行窗口 | `frontend/src/modules/live-run/` |

## 后端目录

后端业务接口按模块拆到：

```text
backend/app/modules/<模块名>/router.py
```

总聚合入口：

```text
backend/app/api/routes.py
```

菜单权限：

```text
backend/app/core/menu.py
```

## Worker 自动化执行

自动化执行侧拆成独立代码侧模块：

```text
worker/app/modules/api_automation/
worker/app/modules/ui_automation/
```

接口自动化技术栈：

```text
Python 3 + pytest + requests + allure
```

UI 自动化技术栈：

```text
Python 3 + pytest + Playwright + allure
```

worker 主入口：

```text
worker/app/worker.py
```

## 改造原则

- 改某个页面长什么样：先找 `frontend/src/modules/<module>/ui/`。
- 改某个页面的保存、删除、执行、统计、筛选：先找 `frontend/src/modules/<module>/feature/`。
- 改通用前端工具：优先改 `frontend/src/shared/`。
- 改接口自动化代码侧能力：优先改 `worker/app/modules/api_automation/`。
- 改 UI 自动化代码侧能力：优先改 `worker/app/modules/ui_automation/`。
- 改菜单权限：优先改 `backend/app/core/menu.py`。
