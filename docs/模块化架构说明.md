# 模块化架构说明

本项目正在按“模块独立、界面和实现分离”的方式组织。

## 前端

前端业务模块目录：

```text
frontend/src/modules/
```

每个模块一个文件夹。模块 UI 放在模块自己的 `XxxPanel.jsx` 中，通用实现放在 `frontend/src/shared/`。

示例：

```text
frontend/src/modules/apiTesting/ApiCasePanel.jsx   接口测试页面 UI
frontend/src/shared/apiClient.js                   前端请求实现
frontend/src/shared/jsonTools.js                   JSON 对比实现
frontend/src/shared/codec.js                       转码实现
```

## Worker 自动化执行

执行侧已经拆成独立模块：

```text
worker/app/modules/api_automation/
worker/app/modules/ui_automation/
```

接口自动化：

```text
Python 3 + pytest + requests + allure
```

UI 自动化：

```text
Python 3 + pytest + Playwright + allure
```

worker 主入口：

```text
worker/app/worker.py
```

它只负责从 Redis 取任务、分发到对应自动化模块、回写数据库。

## 后端

后端当前的业务接口入口仍是：

```text
backend/app/api/routes.py
```

但菜单权限已经独立到：

```text
backend/app/core/menu.py
```

后续如果继续拆后端，推荐拆成：

```text
backend/app/modules/auth/router.py
backend/app/modules/projects/router.py
backend/app/modules/api_testing/router.py
backend/app/modules/ui_testing/router.py
backend/app/modules/file_transfer/router.py
backend/app/modules/image_tools/router.py
backend/app/modules/runs/router.py
backend/app/modules/reports/router.py
backend/app/modules/users/router.py
```

## 改造原则

- 页面 UI 改造：优先改 `frontend/src/modules/<模块名>/`。
- 通用前端功能：优先改 `frontend/src/shared/`。
- 接口自动化代码侧能力：优先改 `worker/app/modules/api_automation/`。
- UI 自动化代码侧能力：优先改 `worker/app/modules/ui_automation/`。
- 菜单权限：优先改 `backend/app/core/menu.py`。
