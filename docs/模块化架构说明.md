# 模块化架构说明

本项目按“菜单模块独立、页面文件和功能文件分离”的方式组织。菜单在页面上仍显示中文，代码目录使用“编号 + 英文”，让文件夹顺序和平台菜单顺序一致，同时减少 Windows、Git、Docker 和服务器脚本里的编码问题。

## 前端目录

前端业务模块目录：

```text
frontend/src/modules/
```

当前结构示例：

```text
frontend/src/modules/
  01-projects/
    ProjectPanel.jsx
    projectFeature.js
    README.md
  01-test-objects/
    TestObjectPanel.jsx
    testObjectFeature.js
    README.md
  02-api-testing/
    ApiCasePanel.jsx
    apiCaseFeature.js
    README.md
  03-ui-testing/
    UiCasePanel.jsx
    uiCaseFeature.js
    README.md
  08-run-history/
    RunsPanel.jsx
    RunDetail.jsx
    runHistoryFeature.js
    README.md
```

每个菜单模块现在只有一层目录：

| 文件类型 | 作用 |
| --- | --- |
| `*Panel.jsx`、`Login.jsx`、`LiveRunWindow.jsx` | 页面组件，主要负责按钮、表格、表单、弹窗、提示文案和页面样式。 |
| `*Feature.js` | 功能实现、数据转换、接口调用、统计、筛选、校验逻辑。 |
| `README.md` | 模块说明和改造入口。 |

前端通用能力统一放在：

```text
frontend/src/shared/
```

例如请求封装、JSON 工具、转码工具、报告导出、文件预览等。多个页面都会用到的能力不要复制到每个模块里，优先放到 `shared`。

## 后端目录

后端业务接口按模块拆到：

```text
backend/app/modules/<模块名>/router.py    接口层：FastAPI 路由、权限、请求响应
backend/app/modules/<模块名>/service.py   业务层：保存、删除、查询、文件处理、报告汇总等
backend/app/modules/<模块名>/schemas.py   模型层：只属于该模块的请求/响应模型
```

总聚合入口：

```text
backend/app/api/routes.py
```

菜单权限：

```text
backend/app/core/menu.py
```

目前已经继续拆出 service/schema 的后端模块：

```text
users/service.py
reports/service.py
test_objects/service.py
test_objects/schemas.py
file_transfer/service.py
image_tools/service.py
image_tools/schemas.py
```

## Worker 自动化执行

自动化执行侧拆成独立代码侧模块：

```text
worker/app/modules/api_automation/
worker/app/modules/ui_automation/
```

接口自动化技术栈：

```text
Python 3 + pytest + requests + allure
```

UI 自动化技术栈：

```text
Python 3 + pytest + Playwright + allure
```

worker 主入口：

```text
worker/app/worker.py
```

## 改造原则

- 改某个页面长什么样：先找 `frontend/src/modules/<module>/*Panel.jsx`。
- 改某个页面的保存、删除、执行、统计、筛选：先找 `frontend/src/modules/<module>/*Feature.js`。
- 改通用前端工具：优先改 `frontend/src/shared/`。
- 改接口自动化代码侧能力：优先改 `worker/app/modules/api_automation/`。
- 改 UI 自动化代码侧能力：优先改 `worker/app/modules/ui_automation/`。
- 改菜单权限：优先改 `backend/app/core/menu.py`。
