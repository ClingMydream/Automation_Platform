# 模块化架构说明

本项目按“菜单模块独立、功能实现和界面展示分离”的方式组织。你打开代码时，可以先按左侧菜单名称找目录。

## 前端目录

前端业务模块目录：

```text
frontend/src/modules/
```

当前结构示例：

```text
frontend/src/modules/
  接口测试/
    接口测试功能/
      README.md
    接口测试UI/
      ApiCasePanel.jsx
      README.md
  UI测试/
    UI测试功能/
      README.md
    UI测试UI/
      UiCasePanel.jsx
      README.md
  执行记录/
    执行记录功能/
      README.md
    执行记录UI/
      RunsPanel.jsx
      RunDetail.jsx
      README.md
```

每个菜单模块都有两个子目录：

| 子目录 | 作用 |
| --- | --- |
| `某某功能/` | 说明这个模块的功能边界、后端接口、执行侧代码入口，方便你改业务逻辑。 |
| `某某UI/` | 放页面组件，主要负责按钮、表格、表单、弹窗、提示文案和页面样式。 |

前端通用能力仍统一放在：

```text
frontend/src/shared/
```

例如请求封装、JSON 工具、转码工具、报告导出、文件预览等。多个页面都会用到的能力不要复制到每个模块里，优先放到 `shared`。

## 后端目录

后端业务接口已经按模块拆到：

```text
backend/app/modules/<模块名>/router.py
```

总聚合入口：

```text
backend/app/api/routes.py
```

菜单权限：

```text
backend/app/core/menu.py
```

当前后端模块：

```text
backend/app/modules/auth/router.py
backend/app/modules/projects/router.py
backend/app/modules/api_testing/router.py
backend/app/modules/ui_testing/router.py
backend/app/modules/file_transfer/router.py
backend/app/modules/image_tools/router.py
backend/app/modules/runs/router.py
backend/app/modules/reports/router.py
backend/app/modules/users/router.py
```

## Worker 自动化执行

自动化执行侧拆成独立代码侧模块：

```text
worker/app/modules/api_automation/
worker/app/modules/ui_automation/
```

接口自动化技术栈：

```text
Python 3 + pytest + requests + allure
```

UI 自动化技术栈：

```text
Python 3 + pytest + Playwright + allure
```

worker 主入口：

```text
worker/app/worker.py
```

它只负责从 Redis 取任务、分发到对应自动化模块、回写数据库。

## 改造原则

- 改某个页面长什么样：先找 `frontend/src/modules/菜单名/菜单名UI/`。
- 改某个功能怎么执行：先看 `frontend/src/modules/菜单名/菜单名功能/README.md`，再按说明去后端或 worker 找实现。
- 改通用前端工具：优先改 `frontend/src/shared/`。
- 改接口自动化代码侧能力：优先改 `worker/app/modules/api_automation/`。
- 改 UI 自动化代码侧能力：优先改 `worker/app/modules/ui_automation/`。
- 改菜单权限：优先改 `backend/app/core/menu.py`。
